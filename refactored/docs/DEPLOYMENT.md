# استقرار (Deployment)

## پیش‌نیازها

- **Node.js ۲۰ یا بالاتر** (با Node 22 تست شده).
- npm (یا bun). هیچ سرویس ابری، دیتابیس یا کلید API لازم نیست.

---

## حالت توسعه

```bash
cd refactored
npm install
npm run dev
```

روی `http://localhost:3000` بالا می‌آید. تغییرات با HMR بازتاب می‌یابند.

برای اجرا روی پورت دلخواه:

```bash
npx vite dev --port 8088 --strictPort
```

---

## ساخت و اجرای پروداکشن

```bash
npm run build                 # خروجی در dist/ (client + server)
node dist/server/server.js
```

نکات:
- متغیر محیطی `PORT` پورت را تعیین می‌کند (پیش‌فرض معمولاً 3000).
- **دایرکتوری کاری (cwd)** مهم است: داده‌ها در `<cwd>/data` نوشته می‌شوند. سرور را همیشه از
  ریشه‌ی پروژه (یا یک مسیر ثابت) اجرا کنید تا محل داده‌ها ثابت بماند.

---

## داده و پایداری

همه‌چیز زیر پوشه‌ی `data/` (نسبت به cwd) ذخیره می‌شود:

```
data/
├── voxelforge.json     # کاربران، نشست‌ها، سفارش‌ها، تنظیمات (نوشتن اتمیک)
├── uploads/<userId>/   # فایل‌های STL
└── receipts/<userId>/  # رسیدهای پرداخت
```

- این پوشه در `.gitignore` است و نباید کامیت شود.
- برای **پشتیبان‌گیری**: کل پوشه‌ی `data/` را کپی کنید، یا از دکمه‌ی «دانلود پشتیبان» در
  `/settings` برای گرفتن دیتابیس JSON استفاده کنید (فایل‌ها جداگانه باید کپی شوند).
- نوشتن روی فایل اتمیک است (فایل موقت + `rename`) تا با قطع برق/کرش خراب نشود.

---

## اجرای دائمی (سرویس)

برای روشن ماندن سرور، از یک مدیر فرایند استفاده کنید، مثلاً PM2:

```bash
npm run build
pm2 start dist/server/server.js --name voxelforge --cwd /path/to/refactored
pm2 save
```

یا یک سرویس systemd بسازید که `WorkingDirectory` آن ریشه‌ی پروژه باشد.

---

## شبکه و آفلاین

- اپ برای کار **بدون اینترنت** طراحی شده؛ تنها وابستگی خارجیِ زمان اجرا، فونت Vazirmatn از
  Google Fonts است (در `__root.tsx`). اگر سرور کاملاً آفلاین است و می‌خواهید فونت هم محلی
  باشد، لینک فونت را با یک نسخه‌ی محلی جایگزین کنید (اختیاری؛ نبودِ فونت فقط ظاهر را تغییر می‌دهد).
- برای دسترسی از شبکه‌ی محلی، با `--host` اجرا کنید یا روی همه‌ی اینترفیس‌ها bind کنید و
  پورت را در فایروال باز کنید.

---

## امنیت در پروداکشن

- پشت یک **reverse proxy با HTTPS** (مثلاً Nginx/Caddy) قرار دهید؛ کوکی نشست `httpOnly` است.
- دسترسی فایل‌سیستمی به پوشه‌ی `data/` را محدود کنید (شامل هش رمزهاست).
- پشتیبان‌های منظم بگیرید.
