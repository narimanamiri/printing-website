# موتور اسلایس (Slicer Engine)

همه‌ی منطق در [`src/lib/stl-parser.ts`](../src/lib/stl-parser.ts). این موتور برخلاف تخمین
ساده‌ی `حجم × چگالی × اینفیل`، وزن را مثل یک اسلایسر واقعی (Cura / PrusaSlicer) **تفکیک‌شده**
محاسبه می‌کند و خروجی‌اش معمولاً تا **±۱۰٪** با وزن واقعی پرینتر می‌خواند.

---

## ۱) پارس هندسه

```ts
parseGeometry(file | ArrayBuffer): Promise<Float32Array>   // ۹ عدد اعشاری به ازای هر مثلث
```

- هم **باینری** و هم **ASCII** را تشخیص می‌دهد (اندازه‌ی بدنه‌ی STL باینری دقیقاً `84 + n×50` بایت است).
- نرمال ذخیره‌شده‌ی فایل **نادیده گرفته می‌شود** (اغلب نامعتبر است)؛ نرمال‌ها هندسی محاسبه می‌شوند.
- مدل فقط **یک‌بار** پارس می‌شود؛ هندسه بین موتور قیمت و پیش‌نمایش سه‌بعدی مشترک است.

---

## ۲) جهت‌گیری (چرخش + مقیاس)

```ts
interface Orient { scale; rotXDeg; rotYDeg; rotZDeg }
applyOrient(positions, orient): Float32Array          // اعمال مقیاس + چرخش اویلر (X→Y→Z)
statsFromGeometry(positions, orient?): StlStats       // محاسبه‌ی آمار با تبدیل، در یک پاس
```

چون چرخش/مقیاس **هندسه را تغییر می‌دهد**، با هر تغییر:
- ابعاد جعبه (bbox)، مساحت سطح و حجم به‌روز می‌شوند،
- این‌که کدام سطوح «اورهنگ» (آویزان) هستند عوض می‌شود → ساپورت و قیمت بازمحاسبه می‌شوند.

مقیاس‌دهی دقیق است: مقیاس ۲× ⇒ حجم ۸×، مساحت ۴×، ابعاد ۲×. (در `scripts/selftest.ts` تست شده.)

---

## ۳) آمار هندسی — `StlStats`

```ts
{ volumeCm3, surfaceAreaCm2, verticalAreaCm2, horizontalAreaCm2, overhangAreaCm2, triangles, bbox }
```

برای هر مثلث (پس از اعمال جهت‌گیری):
- **حجم علامت‌دار** با جمع چهاروجهی‌ها (apex در مبدأ).
- **مساحت و نرمال** از حاصل‌ضرب خارجی دو ضلع.
- دسته‌بندی بر اساس مؤلفه‌ی z نرمال واحد (`nz`):
  - `|nz| ≥ cos45° (۰٫۷۰۷)` → **سطح افقی** (پوسته‌ی رویی/زیری)،
  - در غیر این صورت → **سطح عمودی** (دیواره‌ی محیطی)،
  - `nz < −۰٫۵` (شیب بیش از ۳۰° رو به پایین) → **اورهنگ** (نیازمند ساپورت).
- اگر جهت‌بندی مش معکوس باشد (حجم علامت‌دار منفی)، تشخیص اورهنگ خودکار قرینه می‌شود.

---

## ۴) مدل وزن — `estimatePrint(stats, settings)`

```ts
interface PrintSettings {
  quality; infill; material; support; quantity?;
  pricePerGram?; minOrderToman?; buildVolume?;   // از تنظیمات کسب‌وکار
}
```

مراحل:

1. **ضخامت‌ها**
   - `wallThickness = wallCount × lineWidth`
   - `skinThickness = ((topLayers + bottomLayers) / 2) × layerHeight`
2. **حجم پوسته** = `مساحت‌عمودی × wallThickness + مساحت‌افقی × skinThickness`
   (محدود به حداکثر حجم کل، برای قطعات ریز).
3. **حجم اینفیل** = `(حجم کل − حجم پوسته) × درصد اینفیل`.
4. **حجم ساپورت** (اختیاری) = `مساحت‌اورهنگ × ارتفاع جعبه × ۰٫۵ × ۰٫۱۵`.
5. **وزن** = `(پوسته + اینفیل + ساپورت) × چگالی متریال × تعداد`.

خروجی‌های دیگر:
- **طول فیلامنت (متر)** = `حجم مصرفی ÷ سطح مقطع فیلامنت ۱٫۷۵mm`.
- **زمان چاپ تخمینی** = `حجم مصرفی ÷ شار حجمی ÷ بازده`، که `شار = سرعت × عرض‌خط × ارتفاع‌لایه`.
- **تفکیک وزن**: گرم پوسته / اینفیل / ساپورت.

---

## ۵) قیمت

```
unitCost  = ceil(unitWeight × pricePerGram × ضریبِ متریال)
total     = max(unitCost × تعداد , minOrderToman)
```

- `pricePerGram` و `minOrderToman` از **تنظیمات کسب‌وکار** خوانده می‌شوند (پیش‌فرض ۳۰٬۰۰۰ و ۵۰٬۰۰۰).
- اگر کف حداقل‌سفارش اعمال شود، در رابط کاربری اعلام می‌شود (`minApplied`).

---

## ۶) حجم چاپ

```ts
fitsBuildVolume(bbox, buildVolume?): boolean   // مقایسه‌ی ابعاد مرتب‌شده (اجازه‌ی چرخش)
```

اگر مدل در بستر جا نشود، در صفحه‌ی قیمت هشدار داده می‌شود. پیش‌فرض بستر `۲۵۰×۲۱۰×۲۱۰` میلی‌متر
(قابل تغییر در تنظیمات).

## ثابت‌های ماشین و متریال

```ts
NOZZLE_DIAMETER = 0.4    LINE_WIDTH = 0.42    FILAMENT_DIAMETER = 1.75
PRINT_SPEED = 50 mm/s    TIME_EFFICIENCY = 0.42
SUPPORT_DENSITY = 0.15   SUPPORT_HEIGHT_FRACTION = 0.5
```

| متریال | چگالی (g/cm³) | ضریب قیمت |
|---|---|---|
| PLA | ۱٫۲۴ | ۱٫۰۰ |
| PETG | ۱٫۲۷ | ۱٫۱۵ |
| ABS | ۱٫۰۴ | ۱٫۱۰ |
| TPU | ۱٫۲۱ | ۱٫۶۰ |
| رزین | ۱٫۱۵ | ۱٫۸۰ |

| پریست کیفیت | ارتفاع لایه | دیواره | لایه‌ی رویی | لایه‌ی زیری |
|---|---|---|---|---|
| پیش‌نویس | ۰٫۲۸ | ۲ | ۳ | ۳ |
| استاندارد | ۰٫۲۰ | ۳ | ۴ | ۴ |
| دقیق | ۰٫۱۲ | ۴ | ۵ | ۵ |

> چگالی‌ها و پریست‌ها فیزیکی/استاندارد هستند و در کد تعریف شده‌اند. فقط **قیمت هر گرم**،
> **حداقل سفارش** و **حجم چاپ** از پنل ادمین قابل تغییرند.

## نمونه (راستی‌آزمایی)

مکعب ۲۰ میلی‌متری، استاندارد، اینفیل ۲۰٪، PLA:

```
وزن ≈ ۴٫۶۲ گرم   (پوسته ۳٫۲۹ + اینفیل ۱٫۳۳)
طول فیلامنت ≈ ۱٫۵۵ متر     زمان ≈ ۳۵ دقیقه
اینفیل ۱۰۰٪ ⇒ ۹٫۹۲ گرم ≈ ۸cm³ × ۱٫۲۴ (مکعب توپر)  ✅
```

تست خودکار:

```bash
node --experimental-strip-types scripts/selftest.ts
```

این اسکریپت حجم/مساحت/وزن، مقیاس و چرخش، کف حداقل‌سفارش، حجم چاپ و قیمت قابل‌تنظیم را بررسی می‌کند.

---

## دقت و محدودیت‌ها

- تخمین بر پایه‌ی هندسه است؛ وزن واقعی بسته به ساپورت، رفت/برگشت نازل و تنظیمات دقیق پرینتر
  معمولاً تا **±۱۰٪** فرق می‌کند.
- مش باید **بسته (watertight)** باشد؛ در غیر این صورت حجم معتبر محاسبه نمی‌شود.
- تخمین ساپورت محافظه‌کارانه است؛ زمان چاپ یک تخمین درجه‌اول است (بدون شبیه‌سازی مسیر واقعی).
